## Tutorial 1 — The Complete Guide to Building Skills for Claude

**URL:** https://resources.anthropic.com/hubfs/The-Complete-Guide-to-Building-Skill-for-Claude.pdf  
**Source type:** official (Anthropic)  
**Difficulty:** beginner → advanced  
**Relevance score:** 5 — This is Anthropic's own official 30-page guide covering everything from fundamentals to distribution  
**Skill type potential:** AI/agent pattern, document generation, automation, MCP integration

### Summary

This is the definitive official guide to building Claude skills, published by Anthropic. It covers the complete lifecycle — from planning a skill to distributing it — across 6 chapters: Fundamentals, Planning & Design, Testing & Iteration, Distribution & Sharing, Patterns & Troubleshooting, and Resources.

The guide opens with a strong framing: a skill is "a set of instructions packaged as a simple folder that teaches Claude how to handle specific tasks or workflows." Skills are positioned as the knowledge layer on top of MCP (which provides tool access) — MCP = the kitchen, Skills = the recipes.

Three core use case categories are defined: (1) **Document & Asset Creation** (consistent docs, designs, code), (2) **Workflow Automation** (multi-step MCP-coordinated processes), and (3) **MCP Enhancement** (workflow guidance layered on top of an MCP server).

Technical requirements are strict and exact: `SKILL.md` must be case-sensitive, skill folder names must be kebab-case only, no README.md inside skill folder, no XML tags in frontmatter, and skills named with "claude" or "anthropic" are reserved. The `description` field is the primary trigger mechanism and must include both what the skill does AND when to use it (under 1024 characters).

### Key techniques / patterns

- **Progressive disclosure**: 3 levels — YAML frontmatter (always loaded, ~100 tokens), SKILL.md body (loaded when triggered), bundled resources (loaded on demand). Keep SKILL.md under 5,000 words.
- **5 design patterns**: Sequential workflow orchestration, Multi-MCP coordination, Iterative refinement (quality loops), Context-aware tool selection (decision trees), Domain-specific intelligence (compliance/rules embedded in logic)
- **Testing approach**: Triggering tests (does it load when it should?), Functional tests (does it produce correct outputs?), Performance comparison (with vs. without skill baseline)
- **Distribution**: Host on GitHub with public repo + README (separate from skill folder), upload via Settings > Capabilities > Skills for individual users, organization-wide via admin deployment (shipped Dec 2025), or programmatically via `/v1/skills` API endpoint

### Skill potential

This tutorial is itself the foundation for a **skill-quality-checker** skill — a skill that audits any draft SKILL.md against the official checklist (kebab-case name, description has WHAT + WHEN, no XML tags, under 5000 words, progressive disclosure used, error handling included, examples provided). Could output a scored report and suggest specific fixes.

## Tutorial 2 — 55K-Word Developer Guide for Building Agent Skills

**URL:** https://github.com/anthropics/skills/discussions/435  
**Source type:** community (GitHub Discussion)  
**Difficulty:** advanced  
**Relevance score:** 5 — Production-level patterns not found anywhere else; covers scaling challenges, meta-routers, quality rubrics, token optimization  
**Skill type potential:** AI/agent pattern, automation, workflow orchestration

### Summary

A community-authored 55K-word guide by Yarmoluk, written from experience building 20+ production Agent Skills for business operations. The guide addresses the gap between basic documentation and real-world production challenges. Covers 17 chapters from beginner to expert.

The standout contribution is the **meta-skill router pattern** — a solution to the 30-skill limit problem where you consolidate 14+ sub-skills under a single entry point. The router skill reads user intent and dispatches to the correct sub-skill. This is a genuinely novel architectural pattern not documented in official sources.

A **100-point skill quality rubric** is introduced with weighted dimensions (similar to RAGAS evaluation frameworks), enabling automated quality gating of skill outputs. An accompanying "Skill Quality Analyzer" agent skill is released alongside the guide.

The guide also covers **token efficiency design** with empirical analysis: measures "insight per density token" and "reasoning depth hops before performance collapse." The key finding is that structured instructions decomposed into typed semantic blocks (Role, Objective, Constraints, Examples) with XML-optimized formatting dramatically reduce token waste.

**Session logging patterns** for cross-session continuity are described — essential for long workflows where context fills up. Pipeline orchestration (chaining skills where output feeds next skill) is covered with intermediate state validation and error propagation guidance.

### Key techniques / patterns

- **Meta-skill router**: One entry point skill that routes to 14+ sub-skills, solving the 30-skill limit
- **Token density optimization**: Decompose instructions into typed semantic blocks (Role, Objective, Constraints, Examples)
- **Session logging**: Cross-session state persistence via structured log files
- **Pipeline orchestration**: Skill chaining with intermediate validation between steps
- **100-point quality rubric**: Automated scoring across 7 dimensions for skill quality gating

### Skill potential

Directly actionable for building a **skill-orchestrator** skill — a meta-router that dispatches user requests to the right sub-skill. Also provides the blueprint for a **skill-quality-auditor** skill that scores any SKILL.md against the 100-point rubric and outputs a detailed improvement report.

### Notable snippets

> "Instruction structure matters more than instruction content. Decompose into: Role → Objective → Constraints → Examples. Use XML-optimized formatting. Token waste often comes from unstructured instructions."

### Summary

A curated "awesome list" repository with 11.6k stars documenting the Claude skills ecosystem. Serves as the community's central index for finding, installing, and understanding Claude skills. Launched October 16, 2025 (official skills launch date).

The repo categorizes official skills across Document, Design & Creative, Development, Communication, and Skill Creation domains. Highlights notable community contributions including obra/superpowers (20+ battle-tested skills including TDD and debugging), ios-simulator-skill, playwright-skill, and Trail of Bits security skills.

Key clarification for skill authors: **Skills vs MCP vs Subagents vs Prompts** — Skills = reusable procedural knowledge across conversations; MCP = external data/API integration; Subagents = independent task execution with specific permissions; Prompts = one-time instructions. Getting this distinction right determines whether a skill is the right tool.

Security section is important: "Skills can execute arbitrary code in Claude's environment. Only install skills from trusted sources." Recommends peer review before team distribution, regular audits, and least-privilege access principles.

Installation is now streamlined: Claude Code users run `/plugin marketplace add anthropics/skills`, while Claude.ai users go to Settings > Capabilities > Enable Skills.

### Key techniques / patterns

- Progressive disclosure architecture: metadata (~100 tokens) → full instructions (<5k tokens) → resources (on demand)
- Security-first: treat skills like code dependencies — review before installing
- Version control skills via git like any other code
- Skills ecosystem has 150+ community skills indexed on claudemarketplace.com (as of March 2026)

## Tutorial 4 — Building Agents with the Claude Agent SDK

**URL:** https://claude.com/blog/building-agents-with-the-claude-agent-sdk  
**Source type:** official (Anthropic Engineering Blog)  
**Difficulty:** intermediate  
**Relevance score:** 4 — Covers the agent loop and SDK patterns that skills plug into; essential context for building skills that work in agentic pipelines  
**Skill type potential:** AI/agent pattern, automation, API integration

### Summary

Published September 29, 2025, this post announces the renaming of Claude Code SDK to Claude Agent SDK, signaling broader applicability. Core principle: "give your agents a computer, allowing them to work like humans do."

The **4-phase agent loop** is the central concept: (1) Gather Context — agents fetch/update their own information using agentic search (bash grep/tail), semantic search, or subagents; (2) Take Action — tools, bash scripts, code generation, or MCPs; (3) Verify Work — rules-based checks, visual feedback (screenshots), or LLM-as-judge; (4) Iterate — repeat until quality threshold met.

Key insight for skill authors: **skills are the instruction layer inside the agent loop**. When an agent runs, it loads relevant skills to know *how* to execute each phase. A well-designed skill maps directly to one or more phases of the agent loop and includes verification steps (Phase 3) that most beginner skills skip.

MCPs handle authentication automatically for services like Slack, GitHub, and Asana — skill authors don't need to implement auth, just call the MCP tools correctly. The post emphasizes that "code is precise, composable, and infinitely reusable" — bundling validation scripts in skills is strongly preferred over language-based instructions for critical checks.

### Key techniques / patterns

- Agent loop: Gather Context → Take Action → Verify Work → Iterate
- Agentic search: use bash grep/tail to navigate file systems (faster and more transparent than vector search)
- LLM-as-judge: spawn a secondary evaluator for fuzzy quality criteria (tone, formatting)
- Compaction: automatic context summarization when context window fills
- Bundle validation scripts in skills rather than relying on language instructions for critical checks

### Skill potential

Directly applicable to improving any existing skill by adding a proper **Verify Work** phase with bundled validation scripts. Could inspire a **agent-loop-scaffold** skill — a meta-skill that, when given a use case, generates a complete 4-phase agent loop skill structure with verification scripts as stubs.

### Notable snippets

> "Code is precise, composable, and infinitely reusable. For validation that matters, write a script — don't just say 'validate the data'."

> "Subagents enable parallelization and context isolation — useful for sifting large datasets or running independent checks concurrently."

**Difficulty:** beginner → intermediate

**Skill type potential:** All types

### Summary

The official Anthropic repository housing all example skills and the Agent Skills specification. With 122k stars and 14.2k forks, it's the most starred skills repository in the ecosystem. Contains skill examples organized into Creative & Design, Development & Technical, Enterprise & Communication, and Document Skills categories.

The `/template` folder provides the canonical starting point for any new skill — the minimal structure with YAML frontmatter and a commented instruction scaffold. The `/spec` folder previously contained the full Agent Skills spec but now redirects to agentskills.io/specification (the spec has become an open standard, not just an Anthropic document).

Key partner integration: **Notion Skills for Claude** is listed as an official partner, showing the pattern for third-party service providers to ship skills alongside their MCP servers. Expo (React Native framework) also ships official skills.

Notable community skills highlighted in the repo: Trail of Bits security skills for code vulnerability analysis — a strong example of domain-specific intelligence (Pattern 5 from the official guide) applied to a specialized technical domain.

The document skills (pdf, docx, pptx, xlsx) are source-available rather than open source — meaning they can be used but not forked for commercial redistribution. This is relevant for skill authors building on top of them.

### Key techniques / patterns

- Minimal viable skill: just a folder + `SKILL.md` with name + description frontmatter
- Template available at `/template` folder — use as starting point for every new skill
- `/plugin marketplace add anthropics/skills` is the one-liner for Claude Code installation
- Agent Skills is now an open standard (agentskills.io) — skills should work across Claude and other compatible AI platforms
- Partner skills pattern: ship a skill alongside your MCP server to provide workflow guidance
